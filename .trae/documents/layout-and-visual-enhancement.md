# 荆楚文字 IP 库第二阶段改进计划 (Plan)

## Summary (项目概况)
用户提出需要增加移动端 Tab 栏的实际功能交互，并希望提升页面上“文字列表”的视觉冲击力（让可点击属性一目了然）。同时，需要对首页页面元素的排序进行优化：搜索栏及核心引导区域应当更直接地触达用户，而不是让重要的卡片展示区被其他次要内容过度挤压。

我们将运用 `bolder` (增强视觉冲击力) 技能，通过纹理、光效与排版的重新梳理，打造更具张力的楚文化数字展览体验。

## Current State Analysis (现状分析)
1. **元素排序问题**: 当前 `App.tsx` 中的 DOM 结构为：SearchHero -> 三个介绍文章的 Grid -> CategoryRail (分类标签) -> CharacterCard 网格 -> 底部介绍 Grid。用户反馈搜索框虽在第一屏，但文字卡片展示区（核心内容）被上方的“楚文化溯源”大段介绍挤压，导致用户需要滚动很久才能看到真正的“字”。
2. **Tab 栏交互缺失**: 当前 `MobileTabBar.tsx` 仅有 UI，点击“路线”、“文创”、“我的”没有任何实际反馈，像个“死组件”。
3. **视觉冲击力不足**: 当前的 `CharacterCard.tsx` 虽然有悬停发光，但在静止状态下依然像普通的文字卡片，没有浓厚的“数字艺术品”或“文物”质感，新用户未必能第一眼意识到它们是可交互的深层数字画卷入口。

## Proposed Changes (改进方案)

### 1. 页面元素排序优化 (Layout Reordering)
- **目标文件**: `src/App.tsx`
- **改动说明**: 
  - 将核心的汉字卡片展示区 (`<main>`) 以及分类标签 (`CategoryRail`) **整体上移**，紧跟在 `SearchHero` 下方。
  - 将原本位于中部的“楚文化溯源”（包含“一字一图腾”等两块介绍文章）**下移**至汉字列表之后、Footer 之前。
  - 这样修改后，首屏即是“搜索 + 分类 + 核心大字卡片”，直击主题。

### 2. Tab 栏交互功能 (Tab Bar Interactivity)
- **目标文件**: `src/components/MobileTabBar.tsx`
- **改动说明**: 
  - 为组件添加简单的状态或 Toast 提示层（或简单的弹窗 Alert），当用户点击“路线”、“文创”、“我的”时，弹出优雅的提示框（例如：“荆楚数字路线正在规划中，敬请期待”、“文创商城即将上线”等），而不是毫无反应。这在不引入复杂路由的前提下，给用户提供了明确的操作反馈。

### 3. 增强文字卡片的视觉冲击力与可点击感 (Bolder Visuals)
- **目标文件**: `src/components/CharacterCard.tsx`, `src/index.css`
- **改动说明**: 
  - **加入楚风纹理底纹**: 在卡片内部添加极其微妙的纹理背景（可以使用 CSS 径向渐变或噪点 SVG 模拟漆器表面的光泽），使其看起来像一块数字碑文。
  - **强化可点击提示**: 在卡片的右上角原有的箭头区域增加持续的微弱脉冲动画（Pulse），并在卡片底部或特定位置隐隐透露“点击探秘”的视觉暗示。
  - **大字冲击**: 将卡片中心的核心汉字（`collection-display`）的默认大小再调大 10-20%，并加深字体与背景色的对比，使其如同悬浮在界面之上，带来强烈的压迫感和艺术感。

## Assumptions & Decisions (假设与决策)
- **决策**: 针对 Tab 栏的功能，由于当前架构是单页应用（SPA），引入完整的 `react-router` 成本较高且可能破坏现有的 `url-state` 逻辑。因此，选择“弹窗/Toast 提示”是最快且能提供良好反馈的折中方案。
- **决策**: 卡片的纹理背景将完全通过 CSS (Gradients & Masks) 实现，避免引入外部图片资源导致的加载延迟，保持“纯前端”的轻量化。

## Verification Steps (验证步骤)
1. 刷新页面，验证 `CategoryRail` 和 `CharacterCard` 网格是否紧跟在顶部搜索栏下方，确保用户第一眼就能看到汉字内容。
2. 滚动到底部，确认“楚文化溯源”等长篇介绍文章已成功后置。
3. 缩小窗口至手机视图，点击底部的“路线”、“文创”或“我的”Tab 按钮，验证是否弹出了美观的“敬请期待”提示框。
4. 观察首页的汉字卡片，验证其背景是否多了一层微妙的光影/纹理质感，且中心的大字视觉占比是否明显增大，右上角的箭头是否具备微弱的动态引导效果。
