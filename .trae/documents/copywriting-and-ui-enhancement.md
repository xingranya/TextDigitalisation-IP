# 荆楚文字 IP 库改进计划 (Plan)

## Summary (项目概况)
当前项目已搭建了基本的 Vite + React + Framer Motion 架构，实现了基于 `mock.ts` 的文字网格展示和详情弹窗。但为了适应真实游客扫码导览的场景，当前的文案（如部分“前端架构”、“内容资产层”的开发者口吻文案）显得不够沉浸；同时，作为完整的数字平台，其功能数量、动画微交互以及界面的完整度（如全局 Footer、移动端底部 Tab 栏）都有待进一步提升。

本次改进将应用 `colorize` (色彩策略)、`arrange` (空间布局) 与 `adapt` (响应式与多端适配) 技能，全方位提升真实用户的浏览体验。

## Current State Analysis (现状分析)
1. **文案问题**：`App.tsx` 中底部的三个卡片（“内容资产层”、“交互体验层”、“展陈接入层”）完全是开发者视角的占位文案，对游客毫无意义；部分提示性文案显得过于生硬。
2. **界面与功能缺失**：
   - 缺少全局的底部信息（Footer），网站正规感不足。
   - 缺少移动端常用的底部 Tab 栏导航（首页、文创、我的等），扫码体验不像一个完整的小程序/Web App。
   - 详情页信息层次单薄，缺少：3D/AR文物展示区、语音播报按钮、分享海报入口、相关汉字推荐。
3. **动画与交互**：现有的过渡动画足够平滑，但在列表页滚动、卡片悬停等状态下缺乏细腻的微交互（如发光、涟漪、视差），导致界面“活”的程度不够。
4. **适配与色彩**：页面在移动端的留白和触控目标（Touch Targets）需要按 `adapt` 规范优化；整体色彩系统需要依据 `colorize` 原则进行局部点缀，避免完全黑白的沉闷感。

## Proposed Changes (改进方案)

### 1. 文案重写与内容组织 (Copywriting & Content)
- **目标文件**: `src/App.tsx`
- **改动说明**: 
  - 移除面向开发者的“内容资产层”等说明卡片。
  - 替换为面向游客的“荆楚文化三大脉络”介绍卡片（如：“青铜之光”、“漆木之韵”、“辞赋之美”），文案采用**沉浸文化导览风**，如：“在这片云梦大泽，每一件青铜都回荡着千年的回音...”。
  - 优化搜索栏提示语，如“输入汉字或拼音，寻找你的楚地印记”。

### 2. 功能数量与界面完整度提升 (Feature & UI Completeness)
- **目标文件**: `src/App.tsx`, `src/components/CharacterDetailModal.tsx` (新建或拆分), `src/components/MobileTabBar.tsx` (新建), `src/components/Footer.tsx` (新建)
- **改动说明**:
  - **移动端底部 Tab 栏**: 使用 `adapt` 原则，在移动端 (`md:hidden`) 屏幕底部固定显示类似小程序的 Tab 栏（如：字库、路线、文创），增加触控舒适度。
  - **全局 Footer**: 在 PC 端及页面最底部补充文化机构版权信息、相关链接等，提升正规感。
  - **详情页功能强化 (Detail Modal)**:
    - **语音播报 & 分享**: 在汉字拼音旁增加播放图标；右上角增加“生成分享海报”按钮。
    - **3D/AR 互动占位**: 在“延伸文创 IP”部分，增加一个具有空间感的 3D 模型占位区域，文案提示“点击进入 AR 沉浸观赏”。
    - **相关汉字推荐**: 在详情页底部根据当前汉字的 `category` 筛选出 3-4 个同类汉字进行推荐，引导用户持续探索。

### 3. 动画增强与微交互 (Animation & Micro-interactions)
- **目标文件**: `src/components/CharacterCard.tsx` (新建或拆分), `src/App.tsx`
- **改动说明**:
  - 为汉字卡片引入悬停时的**发光效果**（结合当前字的 `accent` 颜色），并增加轻微的向上浮动与阴影加深。
  - 增加滚动视差：列表项进入视口时不仅有透明度变化，还带有微小的错落视差位移。
  - 详情页内部元素的交错加载（Staggered Children）进一步细化，让释义、3D 占位、相关推荐依次优雅浮现。

### 4. 空间与色彩优化 (Arrange & Colorize)
- **目标文件**: `tailwind.config.js`, 全局组件
- **改动说明**:
  - 应用 `arrange`：优化 Grid 和 Flexbox 的使用，加大区块间的 `gap` 和 `padding`（如使用 `py-16` 到 `py-24` 的跨度），确保数字美术馆的“留白”美学。
  - 应用 `colorize`：利用 `data/characters.ts` 中已有的 `accent` 颜色（如 `oklch(0.62 0.17 25)` 楚红），在卡片的标签、Hover 状态、分享按钮等细节处注入品牌色，打破纯灰白底色的单调。

## Assumptions & Decisions (假设与决策)
- **决策**: 将原先集中在 `App.tsx` 里的长代码拆分为几个核心组件 (`CharacterCard`, `CharacterDetailModal`, `MobileTabBar`, `Footer`)，以便于维护复杂的动画和功能。
- **决策**: 3D/AR 互动和生成海报功能目前只做高完成度的 **UI 占位与交互动画**，不涉及真实的 WebGL 渲染或 Canvas 绘图，以确保纯前端的轻量化和交付速度。
- **假设**: 移动端 Tab 栏的“路线”和“文创”等频道当前可设为开发中状态或提示占位，重点依然是“字库”首页。

## Verification Steps (验证步骤)
1. 检查页面底部的文案是否已全部替换为面向游客的“荆楚文化脉络”导览文案。
2. 缩小浏览器窗口至手机宽度，验证底部是否出现类似小程序的 Tab 栏，且触控区域是否足够大 (min 44px)。
3. 滚动到页面最底部，验证是否存在包含版权和机构信息的 Footer。
4. 点击任意汉字卡片打开详情，验证是否包含：语音播放 Icon、生成海报分享按钮、3D/AR 展品交互区域、以及底部的“相关推荐”列表。
5. 鼠标悬停在首页汉字卡片上，验证是否有基于各自 `accent` 颜色的发光/浮动微交互效果。
